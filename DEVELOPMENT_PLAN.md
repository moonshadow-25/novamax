# NovaMax 开发计划

## 项目概述
NovaMax 是一个本地 AI 模型管理和推理平台，支持多种模型类型（LLM、ComfyUI、TTS、Whisper）的统一管理和调用。

## 已完成的工作

### 1. 项目初始化 ✅
- 创建了前后端项目结构
- 配置了 Vite + React 前端
- 配置了 Express 后端
- 安装了所有依赖包

### 2. 后端核心功能 ✅
- **配置管理** (`backend/src/config/`)
  - 常量定义（端口范围、模型状态等）
  - 配置文件管理器（读取/写入 config.json）

- **模型管理** (`backend/src/services/modelManager.js`)
  - 模型的增删改查
  - 模型状态管理
  - 持久化到 models.json

- **进程管理** (`backend/src/services/processManager.js`)
  - 启动/停止后端进程
  - 端口分配和管理
  - 进程日志收集
  - 支持 LLM (llama.cpp)、ComfyUI、TTS、Whisper

- **API 路由** (`backend/src/routes/`)
  - `/api/models` - 模型管理接口
  - `/api/backends` - 后端进程控制接口
  - `/api/config` - 配置管理接口

### 3. 前端基础框架 ✅
- React + Vite 配置
- 代理配置（指向后端 3001 端口）

### 4. 环境配置 ✅
- 后端端口改为 3001（避免与现有项目冲突）
- llamacpp 路径配置为 `external/llamacpp/vulkan/llama-server.exe`

## 开发建议

1. **优先完成模型管理界面**：这是核心功能，用户需要先能管理模型才能使用其他功能

2. **逐步实现推理功能**：建议按 LLM → TTS → Whisper → ComfyUI 的顺序实现

3. **及时测试**：每完成一个模块就进行测试，确保功能正常

4. **保持简洁**：初期不要过度设计，先实现核心功能

5. **错误处理**：每个 API 调用都要有适当的错误处理
